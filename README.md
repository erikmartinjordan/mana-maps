# Maña Maps

Maña Maps is a static, browser-based map editor for drawing, importing, exporting, and sharing geospatial data. It uses vanilla JavaScript with Leaflet for the 2D map, MapLibre for the 3D globe, and optional Firebase services for accounts and sharing.

Live site: <https://maña.com>

## What it does

- Draw points, lines, polygons, and measurements.
- Edit feature names, geometry, styles, and attributes.
- Import GeoJSON, KML, KMZ, and zipped Shapefiles.
- Export GeoJSON, CSV, KML, KMZ, and Shapefiles.
- Switch between standard map, satellite imagery, and a 3D globe.
- Search places with Nominatim.
- Use the chat panel for map actions and optional AI-assisted commands.
- Save locally in the browser and optionally share maps through Firebase.

## Run locally

This is a static site, so no build step is required.

```bash
git clone https://github.com/erikmartinjordan/mana-maps.git
cd mana-maps
python3 -m http.server 4173
```

Then open <http://127.0.0.1:4173>.

You can also open `index.html` directly, but a local HTTP server better matches production and the E2E test environment.

## Firebase configuration

Firebase config is optional for local UI work. Without it, the editor still loads and local-only features continue to work.

To enable Firebase-backed auth, sharing, tracking, or collaboration locally, create `js/firebase-config.local.js`:

```js
window.MANA_FIREBASE_CONFIGS = {
  pre: { /* Firebase web app config */ },
  pro: { /* Firebase web app config */ }
};
```

Use `pre` for local development and preview environments, and `pro` for production domains. The app chooses `pre` on `localhost`, `127.0.0.1`, `file://`, and `*.local`; otherwise it chooses `pro`. You can override this with `?firebaseEnv=pre` or `?firebaseEnv=pro`.

`js/firebase-config.local.js` is ignored by Git. Production config is generated by the GitHub Pages deployment workflow from repository secrets.

## Tests

The pre-deploy Playwright smoke tests start a local static server and verify the landing page and map editor shell.

```bash
npx playwright test --config=tests/predeploy/playwright.config.js
```

## Tech stack

- Leaflet and Leaflet Draw for 2D mapping and drawing.
- MapLibre GL JS for the 3D globe.
- JSZip and shp.js for import/export helpers.
- Firebase compat SDK for optional auth, Firestore, sharing, tracking, and collaboration.
- Vanilla JavaScript, HTML, and CSS.

## Deployment

The site is deployed as static files to GitHub Pages. The pre-deployment workflow runs validation and smoke tests before deployment.
